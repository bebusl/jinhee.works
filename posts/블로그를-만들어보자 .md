# 왜 만들게 되었나요?

- **왜 굳이 기존 티스토리, 벨로그 같은 걸 안쓰고 직접 만들었나요?**
  대부분 개발자들은 **노션이 실험실 / 작업실**이고,
  **블로그는 외부 전시용 갤러리** 같은 느낌으로 사용해.
  네가 지금 말한 구조는 딱 이런 패턴이야:
  - **평소 정리** → Notion (문서 작성/정리 최강)
  - **대외 공개용** → Remix 블로그 (정제된 글만)

1. 나의 문서화 능력도 보여줄 수 있겠지.
2. 근데 이왕이면 내가 원하는 형태로 보여줄 수 있었음 좋겠다. 싶었음

→ 블로그같은건 제약사항이 생각보다 많아서 내입맛대로 커스텀하려면 직접 만드는 게 베스트.

1. (장) 그리고 노션에 작성하는 글은 md파일로 내려받을 수 있음 → 글을 노션에서 정리 후 md파일만 업로드하면 끝!

   [단] 근데 md 형식을 지원하지 않는 기존 블로그 서비스를 이용할 경우, md파일에서 부분부분 스타일을 변경해줘야 함 → 이 과정을 줄일 수 있음

- 노션 → 중앙 집중화.

- 구현 방향 : 일단 md파일을 직접 읽어서 렌더링하는 방식으로. postgresql을 나중에 붙이자(검색 등의 기능 추가를 위해)

> 2025년 11월 27일 기준 우선 Next.js로 구현 후 Remix로 마이그레이션 하기로 했다.
> Remix 공부에 쏟을 시간이 없음.
