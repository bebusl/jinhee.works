### ✅ 완료된 작업

1. **MDX/MD 파일 렌더링**

### 1. **Frontmatter 활용** (높은 우선순위)

- 코드 블록에 하이라이팅 없음

### 3. **MDX 컴포넌트 추가 스타일링**

- 현재 테이블만 커스텀
- 추가하면 좋은 것들:
    - `h1`, `h2`, `h3` (heading anchor 링크)
    - `code`, `pre` (인라인 코드, 코드 블록)
    - `blockquote` (인용문)
    - `a` (링크 스타일)
    - `img` (이미지 최적화)
    - `ul`, `ol`, `li` (리스트)

### 4. **블로그 목록 페이지**

- 현재 개별 포스트만 렌더링
- `/blog` 페이지에서 전체 포스트 목록 표시
- frontmatter 기반 정렬 (날짜순)

### 5. **Typography 개선**

- `@tailwindcss/typography` 이미 설치됨
- `prose` 클래스만 사용 중
- `prose-lg`, `prose-slate`, dark mode 등 커스터마이징 가능

### 6. **SEO & 메타데이터**

- `generateMetadata` 함수로 동적 메타태그
- OG 이미지
- sitemap.xml, robots.txt

### 7. **기타**

- 목차(TOC) 자동 생성
- 읽는 시간 계산
- 태그/카테고리 시스템
- 이전/다음 포스트 네비게이션
- 코드 복사 버튼

Part 2: 동기화 스크립트 (scripts/sync-notion.mjs)


Node.js ESM 스크립트. GitHub Actions에서 호출.


알고리즘

1. notionClient.databases.query() → 타입=="블로그" AND 상태=="완료" 필터
2. 각 Notion 페이지에 대해:
a. slugify(title) → filename (예: "[리액트-훅-정리.md](http://xn-----hd0jc690myzdspxcnh.md/)")
b. posts/<slug>.md 파일 존재 여부 확인
c. 파일 있으면 gray-matter로 frontmatter 파싱
d. frontmatter.notion_last_edited === page.last_edited_time → skip
e. 변경됨/새파일: n2m.pageToMarkdown([page.id](http://page.id/)) → markdownBody
    - 이미지 블록은 public/images/<notion_id>/ 에 다운로드 후 URL 교체
    f. frontmatter + markdownBody 합쳐서 posts/<slug>.md 작성
3. posts/ 스캔: notion_id 있는데 Notion 응답에 없는 파일 → 삭제
4. 변경 요약 출력 (created/updated/deleted/skipped)

이미지 처리

- Notion 이미지 블록 발견 시 → fetch(url) → public/images/<notion_id>/<filename> 에
저장
- .md 내 URL은 /images/<notion_id>/<filename> 로 교체
- 이미 다운로드된 이미지는 파일 존재 시 스킵

const delay = ms => new Promise(r => setTimeout(r, ms));
